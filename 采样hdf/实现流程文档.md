# 3×3采样训练 + 原始分辨率推理 实现流程文档

## 📋 项目概述

**目标**：通过3×3采样降低训练数据量，加速训练；推理时直接在原始分辨率TIF上进行，输出完整分辨率结果。

**适用场景**：单个tile的时序影像重建

---

## 🎯 核心策略

### 训练阶段
- 使用3×3不重叠窗口采样（取中心点）
- 数据量降低为原来的1/9
- 训练速度提升约9倍

### 推理阶段
- 完全放弃HDF文件
- 直接在原始TIF上使用Window分块读取
- 输出完整分辨率影像
- 只从训练HDF读取标准化参数

## 📐 第一阶段：训练数据准备

### 1.1 影像预处理

**输入**：
- S2时序TIF：`(t, b, h, w)` 例如 `(54, 10, 3001, 3002)`
- S1时序TIF：`(t, b, h, w)` 例如 `(120, 2, 3001, 3002)`

**处理**：
```
步骤1：裁剪到能被3整除
  h_new = (h // 3) * 3  # 3001 -> 3000
  w_new = (w // 3) * 3  # 3002 -> 3000
  
步骤2：3×3采样（取中心点）
  for i in range(0, h_new, 3):
      for j in range(0, w_new, 3):
          sampled[..., i//3, j//3] = original[..., i+1, j+1]
```

**输出**：
- S2采样后：`(54, 10, 1000, 1000)`
- S1采样后：`(120, 2, 1000, 1000)`

### 1.2 同步采样

**必须同步采样的数据**：
1. ✅ S2光学数据
2. ✅ S1雷达数据
3. ✅ 云掩膜（cloud_mask）

**采样规则**：
- 所有数据都取3×3窗口的中心点（索引1）
- 保持空间位置一致性

### 1.3 HDF数据结构

**保存内容**：

## 🔮 第二阶段：推理实现

### 2.1 核心思路

**关键理解**：
- ❌ 不使用训练HDF的X数据
- ✅ 只读取标准化参数（mean, std）
- ✅ 直接从原始TIF读取数据
- ✅ 使用Window分块避免内存溢出

### 2.2 推理流程

#### Step 1: 初始化

**读取标准化参数**：
```
从训练HDF读取：
  - X_mean_40%
  - X_std_40%
  - X_aux_mean
  - X_aux_std
```

**扫描TIF文件**：
```
S2文件夹：
  - S2_L2A_20240902_10m.tif
  - S2_L2A_20240907_10m.tif
  - ...
  
S1文件夹：
  - S1_GRD_20240907.tif
  - S1_GRD_20240912.tif
  - ...
```

#### Step 2: 时间对齐

**提取日期**：
```
S2日期列表：[20240902, 20240907, ...]
S1日期列表：[20240907, 20240912, ...]

转换为DOY：
S2_doy = [245, 250, ...]  # int_to_doy()
S1_doy = [250, 255, ...]
```

**输出时间点**：
```
推理输出：与输入S2的DOY一致
例如：如果输入54个S2时间点，输出也是这54个时间点的重建结果
```

#### Step 3: 分块读取

**Window策略**：
```
BLOCK_SIZE = 512  # 可调整

for i in range(0, height, BLOCK_SIZE):
    for j in range(0, width, BLOCK_SIZE):
        window = Window(j, i, block_w, block_h)
        
        # 读取S2时序的这个block
        s2_block = read_s2_time_series(window)  # (t, b, h, w)
        
        # 读取S1时序的这个block
        s1_block = read_s1_time_series(window)  # (t, b, h, w)
```

#### Step 4: 数据预处理

**4.1 标准化处理**：

S2数据标准化（与训练时完全一致）：
```python
# 步骤1：归一化到0-1范围
s2_normalized = s2_block.astype(np.float32) / 10000

# 步骤2：标准化（使用训练时的mean/std）
s2_normalized = np.where(
    s2_block != 0,  # 只对非0值标准化
    (s2_normalized - X_mean[:, None, None]) / X_std[:, None, None],
    0  # 0值保持为0
)
# 结果：(t, b, h, w) float32，标准化空间
```

S1数据标准化（与训练时完全一致）：
```python
# 步骤1：处理NaN值
s1_block = np.nan_to_num(s1_block, nan=0.0)

# 步骤2：标准化（使用训练时的mean/std）
s1_normalized = (s1_block - X_aux_mean[:, None, None]) / X_aux_std[:, None, None]
s1_normalized = np.where(s1_block != 0, s1_normalized, 0)
# 结果：(t, b, h, w) float32，标准化空间
```

**4.2 Reshape为模型输入格式**：

将空间维度转换为批次维度：
```python
t, b, h, w = s2_normalized.shape
batch_size = h * w  # 每个像素作为一个样本

# (t, b, h, w) -> (h*w, t, b)
# 例如：(54, 10, 512, 512) -> (262144, 54, 10)
s2_input = s2_normalized.reshape(t, b, -1).transpose(2, 0, 1)
s1_input = s1_normalized.reshape(t, b, -1).transpose(2, 0, 1)
```

**4.3 生成Mask（与训练时完全一致）**：

missing_mask（有效观测掩膜）：
```python
# 判断条件：值>0 且 值!=NoData(-32768) 且 值<=10000
valid_mask = (s2_block > 0) & (s2_block != -32768) & (s2_block <= 10000)
valid_mask = valid_mask.all(axis=1)  # 所有波段都有效才算有效
missing_mask = valid_mask.reshape(t, -1).T  # (n, t)
# 0表示缺失，1表示有效观测
```

attention_mask（注意力掩膜）：
```python
# 判断条件：所有波段都是0
attention_mask = (s2_input == 0).all(axis=-1)  # (n, t)
# 0表示有效，1表示缺失（需要注意力机制忽略）
```

#### Step 5: 模型推理

**5.1 构建模型输入**：

将预处理后的数据转换为PyTorch tensor并构建输入字典：
```python
# 转换为tensor并移到GPU
date_input = torch.from_numpy(S2_doy).to(device)  # (t_s2,)
date_input = date_input.unsqueeze(0).repeat(batch_size, 1)  # (n, t_s2)

X = torch.from_numpy(s2_input).to(device)  # (n, t_s2, b_s2)
X_aux = torch.from_numpy(s1_input).to(device)  # (n, t_s1, b_s1)
missing_mask = torch.from_numpy(missing_mask).to(device)  # (n, t_s2)
attention_mask = torch.from_numpy(attention_mask).to(device)  # (n, t_s2)

dates_aux = torch.from_numpy(S1_doy).to(device)  # (t_s1,)
dates_aux = dates_aux.unsqueeze(0).repeat(batch_size, 1)  # (n, t_s1)

# 输出时间点：与输入S2的DOY一致
date_output = date_input.clone()  # (n, t_out)

# 构建输入字典
inputs = {
    "date_input": date_input,      # S2输入时间
    "X": X,                         # S2数据（标准化后）
    "missing_mask": missing_mask,   # 有效观测掩膜
    "X_holdout": X,                 # 原始数据（推理时与X相同）
    "indicating_mask": missing_mask,# 验证掩膜（推理时与missing_mask相同）
    "attention_mask": attention_mask,# 注意力掩膜
    "X_aux": X_aux,                 # S1数据（标准化后）
    "dates_aux": dates_aux,         # S1时间
    "date_output": date_output      # 输出时间点
}
```

**5.2 执行推理**：

```python
with torch.no_grad():
    results = model(inputs, stage="test")
    output = results["reconstructed_data"]  # (n, t_out, b)
```

**关键说明**：
- `stage="test"` 表示推理模式
- 模型输出 `reconstructed_data` 是重建后的完整时序
- 输出时间点与 `date_output` 一致（即输入S2的DOY）

#### Step 6: 后处理

**6.1 理解模型输出**：

⭐ **重要**：模型内部已经完成了反标准化！

模型类 `AnytimeFormer` 中有反标准化方法：
```python
def _reverse_standscale(self, x):
    return x * self.args.X_std + self.args.X_mean
```

因此 `results["reconstructed_data"]` 的值已经是：
- 反标准化后的数据
- 值范围：0-1（归一化范围）
- 数据类型：float32

**6.2 转换为int16**：

只需要乘回scale（10000）：
```python
# 注意：不需要再做 x * std + mean，模型已经做了！
output_int16 = np.round(output * 10000).astype(np.int16)
# 值范围：0-10000
```

**6.3 Reshape回空间维度**：

将批次维度转换回空间维度：
```python
# (n, t_out, b) -> (t_out, b, h, w)
# 例如：(262144, 54, 10) -> (54, 10, 512, 512)

# 步骤1：reshape为4D
output_spatial = output_int16.reshape(block_h, block_w, t_out, b)
# 结果：(h, w, t_out, b)

# 步骤2：调整维度顺序
output_spatial = output_spatial.transpose(2, 3, 0, 1)
# 结果：(t_out, b, h, w)
```

**6.4 写入输出数组**：

将每个时间点的数据写入对应位置：
```python
for t_idx, doy in enumerate(output_doy):
    # 写入这个block的数据到完整影像的对应位置
    output_arrays[doy][:, i:i+block_h, j:j+block_w] = output_spatial[t_idx]
    # output_arrays[doy]: (b, H, W) - 完整影像
    # output_spatial[t_idx]: (b, h, w) - 当前block
```

**6.5 内存管理**：

及时释放不再使用的变量：
```python
del s2_block, s1_block, s2_input, s1_input
del missing_mask, attention_mask, output, output_spatial
gc.collect()  # 每处理10个blocks执行一次垃圾回收
```

#### Step 7: 输出TIF

**写入文件**：
```
for doy in output_doy:
    ymd = doy_to_ymd(year, doy)  # 245 -> 20240902
    filename = f"S2_L2A_reconstructed_{ymd}.tif"
    
    # 使用rasterio写入，保留地理信息
    with rasterio.open(filename, 'w', **profile) as dst:
        dst.write(output_arrays[doy])
```

## 🔧 关键技术点

### 3.1 边界处理

**训练时**：
- 裁剪到能被3整除
- 丢失少量边缘像素（可接受）
- 例如：3001×3002 → 3000×3000（丢失1-2像素）

**推理时**：
- 处理完整影像（包括边缘）
- 边缘区域模型没见过，效果可能稍差
- 影响很小，可接受

### 3.2 NoData/云处理

**问题**：推理时原始TIF可能有云（值为0或NoData）

**解决方案**：
```
方案1：直接填充0（推荐）✅
  - 与训练时逻辑一致
  - 模型已经学会处理缺失值
  - 简单高效

方案2：插值填充（不推荐）
  - 时间维度插值
  - 空间维度插值
  - 复杂，不推荐
```

**实现细节**：
```python
# 标准化时，0值保持为0
s2_normalized = np.where(
    s2_data != 0,  # 只对非0值标准化
    (s2_data / 10000 - X_mean) / X_std,
    0  # 0值保持为0
)

# missing_mask标记哪些是有效观测
missing_mask = (s2_block > 0) & (s2_block != -32768) & (s2_block <= 10000)

# attention_mask标记哪些需要模型忽略
attention_mask = (s2_input == 0).all(axis=-1)  # 所有波段都是0
```

### 3.3 内存优化

**策略**：
1. **分块大小可调**：512×512 或 1024×1024
2. **及时释放**：每个block处理完立即释放
3. **数据类型**：推理用float32，输出转int16
4. **不加载完整时序**：使用Window逐块读取

**内存估算**：
```
单个block内存占用（512×512）：
  S2读取: 54 × 10 × 512 × 512 × 2 bytes (int16) ≈ 280 MB
  S1读取: 120 × 2 × 512 × 512 × 2 bytes (int16) ≈ 125 MB
  S2标准化: 54 × 10 × 512 × 512 × 4 bytes (float32) ≈ 560 MB
  S1标准化: 120 × 2 × 512 × 512 × 4 bytes (float32) ≈ 250 MB
  模型输出: 54 × 10 × 512 × 512 × 4 bytes (float32) ≈ 560 MB
  总计：约 1.8 GB（可接受）

完整影像内存占用（3000×3000，不推荐）：
  S2读取: 54 × 10 × 3000 × 3000 × 2 bytes ≈ 9.7 GB
  总计：约 20+ GB（内存溢出风险）
```

**优化技巧**：
```python
# 1. 及时释放变量
del s2_block, s1_block, output
gc.collect()

# 2. 每处理N个blocks执行一次垃圾回收
if block_idx % 10 == 0:
    gc.collect()

# 3. 使用生成器而不是列表
for s2_path in self.s2_file_paths:  # 逐个读取
    with rasterio.open(s2_path) as src:
        data = src.read(window=window)
        # 处理完立即释放
```

### 3.4 时间序列长度

**训练时**：
- 固定长度（例如54个S2时间点）
- HDF中所有样本长度一致

**推理时**：
- 可能不同tile有不同数量
- 模型支持可变长度（Transformer架构）
- 无需特殊处理，直接输入即可

**注意事项**：
- 输出时间点数 = 输入S2时间点数
- 如果输入30个时间点，输出也是30个
- 如果输入54个时间点，输出也是54个

### 3.5 标准化参数的重要性

**为什么必须使用训练时的标准化参数？**

训练时的标准化：
```python
X_train_normalized = (X_train / 10000 - X_mean_train) / X_std_train
# 模型学习的是这个分布
```

推理时如果用不同的参数：
```python
X_test_normalized = (X_test / 10000 - X_mean_test) / X_std_test
# ❌ 分布不一致，模型会输出错误结果！
```

正确做法：
```python
# 从训练HDF读取标准化参数
with h5py.File(train_hdf_path, "r") as hf:
    X_mean = hf["data"]["X_mean_40%"][:]  # 训练时的均值
    X_std = hf["data"]["X_std_40%"][:]    # 训练时的标准差

# 推理时使用相同参数
X_test_normalized = (X_test / 10000 - X_mean) / X_std
# ✅ 分布一致，模型输出正确
```

### 3.6 模型内部的反标准化

**重要理解**：模型会自动反标准化输出！

模型类中的方法：
```python
class AnytimeFormer:
    def _standscale(self, x):
        """标准化（模型内部可能使用）"""
        return (x - self.args.X_mean) / self.args.X_std
    
    def _reverse_standscale(self, x):
        """反标准化（模型返回前自动调用）"""
        return x * self.args.X_std + self.args.X_mean
```

因此：
```python
# 模型推理
results = model(inputs, stage="test")
output = results["reconstructed_data"]

# ⭐ 这个output已经是反标准化后的！
# 值范围：0-1（归一化范围）
# 不需要再做 output * std + mean

# 只需要乘回scale
output_int16 = np.round(output * 10000).astype(np.int16)
```

**常见错误**：
```python
# ❌ 错误：重复反标准化
output_denorm = output * X_std + X_mean  # 模型已经做了！
output_int16 = np.round(output_denorm * 10000).astype(np.int16)
# 结果：数值错误，可能超出0-10000范围

# ✅ 正确：直接乘回scale
output_int16 = np.round(output * 10000).astype(np.int16)
```

## 📊 完整数据流

### 训练流程
```
原始TIF (3001×3002, int16, 0-10000)
  ↓ 裁剪到能被3整除
裁剪后 (3000×3000)
  ↓ 3×3采样（取中心点）
采样后 (1000×1000)
  ↓ 标准化：÷10000 → (x-mean)/std
标准化后 (float32, 标准化空间)
  ↓ 写入HDF
训练HDF (包含采样数据 + 标准化参数)
  ↓ DataLoader随机读取
训练批次 (batch_size, t, b)
  ↓ 模型训练
训练好的模型.ckpt
```

### 推理流程（详细数据转换）
```
┌─────────────────────────────────────────────────────────┐
│ 1. 原始TIF文件 (3000×3000)                              │
│    数据类型: int16                                       │
│    值范围: 0-10000                                       │
│    维度: (t, b, H, W) 例如 (54, 10, 3000, 3000)        │
└─────────────────────────────────────────────────────────┘
                    ↓ Window分块读取
┌─────────────────────────────────────────────────────────┐
│ 2. Block数据 (512×512)                                  │
│    数据类型: int16                                       │
│    值范围: 0-10000                                       │
│    维度: (t, b, h, w) 例如 (54, 10, 512, 512)          │
└─────────────────────────────────────────────────────────┘
                    ↓ ÷ 10000 (归一化)
┌─────────────────────────────────────────────────────────┐
│ 3. 归一化后                                              │
│    数据类型: float32                                     │
│    值范围: 0-1                                           │
│    维度: (t, b, h, w)                                    │
└─────────────────────────────────────────────────────────┘
                    ↓ (x - mean) / std (标准化)
┌─────────────────────────────────────────────────────────┐
│ 4. 标准化后（预处理完成）                                │
│    数据类型: float32                                     │
│    值范围: 标准化空间（均值0，标准差1）                   │
│    维度: (t, b, h, w)                                    │
└─────────────────────────────────────────────────────────┘
                    ↓ Reshape: (t,b,h,w) → (h*w,t,b)
┌─────────────────────────────────────────────────────────┐
│ 5. 模型输入格式                                          │
│    数据类型: torch.Tensor (float32)                      │
│    值范围: 标准化空间                                     │
│    维度: (n, t, b) 例如 (262144, 54, 10)               │
│    说明: n = h*w，每个像素作为一个样本                   │
└─────────────────────────────────────────────────────────┘
                    ↓ 模型推理 (stage="test")
┌─────────────────────────────────────────────────────────┐
│ 6. 模型输出（标准化空间）                                │
│    数据类型: torch.Tensor (float32)                      │
│    值范围: 标准化空间                                     │
│    维度: (n, t_out, b)                                   │
└─────────────────────────────────────────────────────────┘
                    ↓ 模型内部自动执行: x * std + mean
┌─────────────────────────────────────────────────────────┐
│ 7. 反标准化后（模型返回值）                              │
│    数据类型: torch.Tensor (float32)                      │
│    值范围: 0-1                                           │
│    维度: (n, t_out, b)                                   │
│    ⭐ results["reconstructed_data"] 就是这个状态！       │
└─────────────────────────────────────────────────────────┘
                    ↓ × 10000 (转回原始范围)
┌─────────────────────────────────────────────────────────┐
│ 8. 转换为int16                                           │
│    数据类型: numpy.ndarray (int16)                       │
│    值范围: 0-10000                                       │
│    维度: (n, t_out, b)                                   │
└─────────────────────────────────────────────────────────┘
                    ↓ Reshape: (n,t,b) → (t,b,h,w)
┌─────────────────────────────────────────────────────────┐
│ 9. 空间维度恢复                                          │
│    数据类型: int16                                       │
│    值范围: 0-10000                                       │
│    维度: (t_out, b, h, w) 例如 (54, 10, 512, 512)      │
└─────────────────────────────────────────────────────────┘
                    ↓ 拼接所有blocks
┌─────────────────────────────────────────────────────────┐
│ 10. 完整分辨率输出                                       │
│     数据类型: int16                                      │
│     值范围: 0-10000                                      │
│     维度: (t_out, b, H, W) 例如 (54, 10, 3000, 3000)   │
└─────────────────────────────────────────────────────────┘
                    ↓ 写入TIF文件
┌─────────────────────────────────────────────────────────┐
│ 11. 输出TIF文件                                          │
│     每个时间点一个文件                                    │
│     S2_L2A_reconstructed_20240902.tif                   │
│     S2_L2A_reconstructed_20240907.tif                   │
│     ...                                                  │
└─────────────────────────────────────────────────────────┘
```

## 📝 注意事项

### 数据一致性
1. **空间位置一致**：确保S2、S1、Mask的采样位置完全一致
2. **时间对齐**：S2和S1的DOY必须正确对应
3. **标准化参数**：推理时必须使用训练时的参数（从HDF读取）

### 标准化流程
1. **训练时**：
   - 计算mean/std → 保存到HDF
   - 标准化数据：`(X/10000 - mean) / std`
   - 模型学习标准化空间的分布

2. **推理时**：
   - 从HDF读取mean/std（必须与训练时相同）
   - 标准化数据：`(X/10000 - mean) / std`（相同公式）
   - 模型输出已反标准化（0-1范围）
   - 只需乘回10000转为int16

### 模型输出理解
1. **模型内部自动反标准化**：
   - `results["reconstructed_data"]` 已经是反标准化后的
   - 值范围：0-1（归一化范围）
   - 不需要再做 `output * std + mean`

2. **后处理只需**：
   - 乘回scale：`output * 10000`
   - 转为int16：`astype(np.int16)`
   - Reshape回空间维度

### NoData处理
1. **直接填充0**：与训练逻辑一致，模型已学会处理
2. **不要插值**：增加复杂度，效果未必更好
3. **Mask标记**：missing_mask和attention_mask会告诉模型哪些是缺失值

### 内存管理
1. **合理设置BLOCK_SIZE**：
   - 512×512：约1.8GB内存
   - 1024×1024：约7GB内存
   - 根据GPU/CPU内存调整

2. **及时释放内存**：
   - 每个block处理完立即 `del` 变量
   - 定期执行 `gc.collect()`

3. **避免内存泄漏**：
   - 不要在循环中累积大数组
   - 使用生成器而不是列表

### 地理信息
1. **输出TIF必须保留**：
   - 坐标系统（CRS）
   - 仿射变换（transform）
   - 波段数量和顺序

2. **从参考影像读取**：
   ```python
   with rasterio.open(ref_tif_path) as src:
       profile = src.profile.copy()
       transform = src.transform
       crs = src.crs
   ```

### 边界处理
1. **最后一个block可能不足BLOCK_SIZE**：
   ```python
   block_h = min(BLOCK_SIZE, height - i)
   block_w = min(BLOCK_SIZE, width - j)
   ```

2. **自动处理**：代码已实现边界检查，无需手动调整

### 时间对齐
1. **S2和S1时间点不同**：
   - S2：54个时间点
   - S1：120个时间点
   - 模型会自动对齐（Transformer的优势）

2. **输出时间点**：
   - 默认：与输入S2的DOY一致
   - 可自定义：修改 `date_output` 参数

### 文件命名
1. **TIF文件名格式**：
   - 必须包含日期信息
   - 例如：`S2_L2A_20240902_10m.tif`
   - 代码会自动提取日期

2. **输出文件名**：
   - `S2_L2A_reconstructed_20240902.tif`
   - 保持与输入一致的命名规则

### 调试建议
1. **先测试小数据**：
   - 使用小的BLOCK_SIZE（如256×256）
   - 只处理几个blocks
   - 验证逻辑正确

2. **检查中间结果**：
   - 打印数据形状和值范围
   - 验证标准化参数是否正确
   - 检查mask生成是否合理

3. **对比训练代码**：
   - 确保预处理逻辑一致
   - 确保模型输入格式一致
   - 确保后处理逻辑一致

## 🐛 常见问题排查

### 问题1：内存溢出（OOM）

**症状**：
```
RuntimeError: CUDA out of memory
或
MemoryError: Unable to allocate array
```

**解决方案**：
1. 减小 `block_size`：512 → 256
2. 减小 `inference_batch_size`
3. 使用CPU推理（如果GPU内存不足）
4. 检查是否有内存泄漏

### 问题2：数值范围错误

**症状**：
```
输出影像值超出0-10000范围
或出现负值
```

**原因**：
- 重复反标准化（模型已经做了）

**解决方案**：
- 检查 `_postprocess_block()` 函数
- 确保只做 `output * 10000`
- 不要再做 `output * std + mean`

### 问题3：输出影像全黑或全白

**症状**：
```
输出影像所有像素值都是0或10000
```

**原因**：
- 标准化参数错误
- 模型加载失败
- 输入数据格式错误

**解决方案**：
1. 检查标准化参数是否从正确的HDF读取
2. 检查模型路径是否正确
3. 打印中间结果验证数据流

### 问题4：地理坐标丢失

**症状**：
```
输出TIF无法在GIS软件中正确显示
```

**解决方案**：
- 确保从参考影像读取了 `profile`, `transform`, `crs`
- 检查写入TIF时是否传递了这些参数

### 问题5：推理速度慢

**优化方案**：
1. 增大 `block_size`（如果内存允许）
2. 使用GPU推理
3. 减少时间点数量（如果可以）
4. 使用多进程处理多个tiles
